# P-001: 時計画面 E2Eテスト仕様書

## テスト対象
24時間回転文字盤時計のメイン画面

## テストケース

### TC-001: Canvas初期描画
- **前提条件**: アプリ起動
- **手順**: ウィンドウが表示されるまで待機
- **期待結果**: Canvas要素が表示され、時計が描画されている

### TC-002: 文字盤の数字表示
- **前提条件**: アプリ起動済み
- **手順**: Canvas上の文字盤を確認
- **期待結果**: 0〜23の数字が円周上に表示されている

### TC-003: デジタル時刻表示
- **前提条件**: アプリ起動済み
- **手順**: 針上のデジタル表示を確認
- **期待結果**: MM:SS形式で現在の分・秒が表示されている

### TC-004: 文字盤の回転
- **前提条件**: アプリ起動済み
- **手順**: 数秒間待機しCanvas描画を比較
- **期待結果**: 文字盤が反時計回りに滑らかに回転している

### TC-005: 固定針
- **前提条件**: アプリ起動済み
- **手順**: 針の位置を確認
- **期待結果**: 針が12時方向（上部）に固定されている

### TC-006: ウィンドウリサイズ
- **前提条件**: アプリ起動済み
- **手順**: ウィンドウサイズを変更（拡大・縮小）
- **期待結果**: 時計全体がウィンドウサイズに追従してスケーリングされる

### TC-007: 最小ウィンドウサイズ
- **前提条件**: アプリ起動済み
- **手順**: ウィンドウを200x200まで縮小
- **期待結果**: 200x200以下にはならず、時計が正常に表示される

### TC-008: Retina対応
- **前提条件**: 高DPIディスプレイでアプリ起動
- **手順**: 文字盤の描画品質を確認
- **期待結果**: ぼやけることなく鮮明に描画される

### TC-009: アニメーションフレームレート
- **前提条件**: アプリ起動済み
- **手順**: 60秒間のフレームレートを計測
- **期待結果**: 60fps以上を維持
